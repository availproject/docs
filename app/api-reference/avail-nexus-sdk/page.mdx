# Avail Nexus

## Prerequisites

<Callout type="info">
**BEFORE WE START**

1. The Nexus SDK requires an injected wallet provider (e.g. MetaMask, WalletConnect, etc.)
2. It is designed for client-side browser environments and **cannot be used** in server-side environments.
3. GitHub: [`availproject/nexus-sdk`](https://github.com/availproject/nexus-sdk)
4. `npm.js` Package: [`@avail-project/nexus`](https://www.npmjs.com/package/@avail-project/nexus)
</Callout>

## Dev Environment Setup

<Steps>

### Set Up a Frontend environment

The Avail Nexus SDK requires a wallet provider to be injected into the browser environment.
If your frontend can do that, it's good enough.

### Install the  SDK

Install the Nexus SDK.

Use any package manager of your choice to install the Nexus SDK:

<Tabs items={['npm', 'yarn', 'pnpm']}>

<Tabs.Tab>
```bash filename="Terminal"
npm install @avail-project/nexus
```
</Tabs.Tab>


<Tabs.Tab>
```bash filename="Terminal"
yarn add @avail-project/nexus
```
</Tabs.Tab>


<Tabs.Tab>
```bash  filename="Terminal"
pnpm add @avail-project/nexus
```
</Tabs.Tab>

</Tabs>

### Core SDK vs UI Widgets

The Nexus SDK provides optimized entrypoints for the two different use cases:

1. **Core SDK**: A headless version of the SDK that allows devs to use Avail Nexus without
a React-based environment.
2. **UI Widgets**: A set of React components that make it easier to integrate Avail Nexus into React-based environments.

The following pages will go through both use cases.
