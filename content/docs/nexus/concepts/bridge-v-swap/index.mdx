Nexus allows you to query 2 types of balances for different use cases. These balances determine the sourcing of liquidity when doing certain operations. 

- Bridge Balance
- Swap Balance

## Bridge Balance
Use [Fetch Bridge Balance](/docs/nexus/nexus-sdk/reference/bridge-methods/fetch-bridge-balances) (previously Unified Balance) to get all token balances available for bridge operations across supported chains in a single call. Any Bridge Method you call will source liquidity from these balances.

```jsx
getBalancesForBridge(): Promise<UserAsset[]>
```

## Swap Balance
Use [Fetch Swap Balance](/docs/nexus/nexus-sdk/reference/swap-methods/fetch-swap-balances) to get all token balances available for swap operations across supported chains in a single call. Any Swap Method you call will source liquidity from these balances.

```jsx
getBalancesForSwap(): Promise<Array<{
    amount: string;
    chainID: number;
    decimals: number;
    symbol: string;
    tokenAddress: `0x${string}`;
    universe: Universe;
    value: number;
    logo: string;
}>>
```

## Why two different balances?
Nexus solvers only manage liquidity for canonical bridge tokens (currently ETH, USDC, USDT). Bridge balances reflect what solvers can move across chains directly.

For crosschain swaps, Nexus orchestrates the intent but the actual swaps are executed by partner DEX aggregators. This means swap operations can access a much wider range of tokens, since they aren't limited by solver liquidity.

Because the available tokens differ between bridging and swapping, each operation needs its own balance query — your "swappable" balances are not the same as your "bridgeable" assets.
