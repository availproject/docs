Nexus lets your app move assets and execute transactions across chains without requiring users to bridge manually. Your users get a single-balance experience — your app handles the cross-chain complexity under the hood.

```ts
// 1. Initialize
const sdk = new NexusSDK({ network: 'mainnet' });

// 2. See balances across all chains
const bridgeBalances = await sdk.getBalancesForBridge();
// → Returns tokens available for bridging across chains

const swapBalances = await sdk.getBalancesForSwap();
// → Returns tokens available for cross-chain swaps

// 3. Bridge to any chain — Nexus finds the best route
await sdk.bridge({ token: 'USDC', amount: 100, toChain: 'arbitrum' });

// 4. Or swap any token → any token, cross-chain
await sdk.swap({ from: 'USDC', to: 'ETH', amount: 50, toChain: 'base' });
```

## What Changes for Your Users

| Without Nexus | With Nexus |
|---|---|
| User bridges manually, switches chains, manages gas per chain | User signs once — Nexus routes funds across chains |
| Dev writes bridge logic per chain pair | Dev calls a single SDK method with token + destination |
| Balances scattered across 12 chains | One unified balance |

## Prerequisites

Node.js 18+, a package manager (npm/pnpm/yarn), and an EVM wallet (MetaMask, WalletConnect, etc.). The Nexus SDK is client-side only — it cannot run in server-side environments.

## How Nexus Works

When a user initiates a cross-chain action, Nexus handles the entire flow behind a single signature:

<Steps>

### User Expresses an Intent
The user says "Move 100 USDC to Arbitrum" — your app creates an [intent](/docs/nexus/concepts/intent), a structured declaration of the desired outcome.

### Protocol Collects Funds
Using pre-approved [allowances](/docs/nexus/concepts/allowances), Nexus accesses the user's funds across source chains and deposits them into vault contracts.

### Solvers Compete
[Solvers](/docs/nexus/concepts/solvers) — liquidity providers — race to fulfill the intent by providing funds on the destination chain. Competition keeps fees low.

### Funds Arrive
The user receives tokens on the destination chain. Settlement with the solver happens asynchronously in the background.

</Steps>

## Supported Chains & Tokens

Nexus currently supports bridging and swaps across major EVM chains including Ethereum, Arbitrum, Base, Optimism, and more. Bridge operations support canonical tokens (ETH, USDC, USDT), while swap operations support a broader range of tokens through DEX aggregator partners.

<Callout type="info">
For the full list of supported chains and tokens, see the [SDK API Reference](/docs/nexus/nexus-sdk/quickstart).
</Callout>
