The fastest way to start building with the Nexus SDK is the `create-liquid-apps` starter kit. It scaffolds a complete project with your chosen framework, wallet provider, and a pre-configured headless Nexus SDK setup, all in a single command.
<div className="mt-4" />
By the end of this guide you will have a running app that initializes the Nexus SDK with a connected wallet and fetches unified cross-chain balances.

## Prerequisites

You will need a **WalletConnect Project ID**. Get one free at [cloud.walletconnect.com](https://cloud.walletconnect.com).

## Step 1 — Scaffold your project

<div className="mt-4" />

- Run the following command to scaffold a Nexus SDK app:

```bash
npx create-liquid-apps my-nexus-app --framework next --widgets nexus-core --auth wagmi-familyconnect
```

<div className="mt-4" />

This will create a Next.js project with the headless Nexus SDK (`@avail-project/nexus-core`) and Wagmi + ConnectKit for wallet connections.

<Callout type="info" title="Prefer React + Vite?">
Replace `--framework next` with `--framework react-vite` to scaffold a lightweight Vite boilerplate instead.
</Callout>

<Callout type="info" title="Interactive mode">
You can also run `pnpx create-liquid-apps` without any flags. The CLI will walk you through each option with interactive prompts. When asked for the Nexus flavor, select **Nexus Core**.
</Callout>

<div className="mt-4" />

The CLI will clone the template, install all dependencies, and initialize a git repository. When it finishes you will see:

```bash
Success! Created my-nexus-app using next.
```

## Step 2 — Configure environment variables

<div className="mt-4" />

- Move into your project and set up the environment file:

```bash
cd my-nexus-app
cp .env.local.example .env.local
```
<div className="mt-4" />

- Open `.env.local` and add your WalletConnect Project ID:

```bash title=".env.local"
NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID=your_project_id_here
```

<Callout type="warning" title="Don't skip this step">
Without a valid WalletConnect Project ID, wallet connection will fail silently. Get one for free at [cloud.walletconnect.com](https://cloud.walletconnect.com).
</Callout>

## Step 3 — Start the development server

```bash
npm run dev
```
<div className="mt-4" />

Open [http://localhost:3000](http://localhost:3000) in your browser. You should see the starter app with a **Connect Wallet** button.

## Step 4 — Connect and explore

1. Click **Connect Wallet** and connect using MetaMask, Coinbase Wallet, or any WalletConnect-compatible wallet.
2. Once connected, the SDK initializes automatically with the wallet's EIP-1193 provider.
3. The app fetches and displays your **Unified Balances** — an aggregated view of your tokens across all supported chains.

<Callout type="info" title="What you are seeing">
The scaffolded app comes pre-wired with the headless Nexus SDK (`@avail-project/nexus-core`). Unlike Nexus Elements, there are no pre-built UI components — the SDK gives you full control to build your own interface on top of cross-chain operations like bridging, swapping, and balance fetching.
</Callout>

## What's in the project

Here is an overview of the key files the CLI generated:

```ascii
my-nexus-app/
├── app/
│   ├── layout.tsx          # Root layout with Web3Provider
│   └── page.tsx            # Main page with SDK usage examples
├── components/
│   └── nexus/
│         └── NexusProvider.tsx   # Nexus SDK context provider
├── lib/
│   └── nexus.ts            # SDK instance and helper functions
├── providers/
│   └── Web3Provider.tsx    # Wagmi + ConnectKit + NexusProvider setup
└── .env.local.example      # Environment variable template
```

The provider nesting is already configured for you in `providers/Web3Provider.tsx`:

`WagmiProvider` → `QueryClientProvider` → `NexusProvider` → `ConnectKitProvider` → `{children}`

## Troubleshooting

### 1. Wallet connection fails

1. Verify your WalletConnect Project ID is set correctly in `.env.local`.
2. Make sure you are using `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID` as the variable name.
3. Restart the dev server after changing environment variables.

### 2. Dependencies fail to install

1. Make sure you are running Node.js 18 or later: `node --version`.
2. Delete `node_modules` and the lockfile, then reinstall: `rm -rf node_modules pnpm-lock.yaml && pnpm install`.
3. If using bun, ensure version 1.1.0+: `bun --version`.

### 3. SDK initialization fails

1. Confirm a wallet is connected before the SDK attempts to initialize — the `initialize()` call requires an active EIP-1193 provider.
2. Check `NexusProvider.tsx` to ensure `initializeNexus()` is being called after wallet connection.
3. Check the browser console for errors related to missing provider or network issues.
