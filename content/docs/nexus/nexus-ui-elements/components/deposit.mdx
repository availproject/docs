
## Usage

Wrap your app with the Nexus provider first. See [NexusProvider Setup](/docs/nexus/nexus-ui-elements/get-started/nexus-provider-setup) for provider setup.

Deposit takes a single `config.deposit` block describing the destination asset and the app call to run after Nexus delivers funds.

```tsx showLineNumbers
import { NexusOne } from "@/components/nexus-one/nexus-one";
import { encodeFunctionData } from "viem";

const AAVE_POOL_ARBITRUM = "0x794a61358D6845594F94dc1DB02A252b5b4814aD";
const USDT_ARBITRUM = "0xFd086bC7CD5C481DCC9C85ebE478A1C0b69FCbb9";

const AAVE_ABI = [
  {
    inputs: [
      { internalType: "address", name: "asset", type: "address" },
      { internalType: "uint256", name: "amount", type: "uint256" },
      { internalType: "address", name: "onBehalfOf", type: "address" },
      { internalType: "uint16", name: "referralCode", type: "uint16" },
    ],
    name: "supply",
    outputs: [],
    stateMutability: "nonpayable",
    type: "function",
  },
] as const;

export function DepositExample({ address }: { address?: `0x${string}` }) {
  return (
    <NexusOne
      connectedAddress={address}
      config={{
        mode: "deposit",
        prefill: {
          amount: "0.1",
        },
        deposit: {
          chainId: 42161,
          tokenSymbol: "USDT",
          tokenDecimals: 6,
          tokenAddress: USDT_ARBITRUM,
          title: "Aave",
          protocol: "Aave",
          executeDeposit: (_tokenSymbol, tokenAddress, amount, _chainId, user) => ({
            to: AAVE_POOL_ARBITRUM,
            data: encodeFunctionData({
              abi: AAVE_ABI,
              functionName: "supply",
              args: [tokenAddress, amount, user, 0],
            }),
            gas: 300000n,
            tokenApproval: {
              toTokenAddress: tokenAddress,
              amount,
              spender: AAVE_POOL_ARBITRUM,
            },
          }),
        },
      }}
    />
  );
}
```

## Configuration

Deposit uses only `mode: "deposit"`. This mode is exact-out: users enter the destination deposit amount, pay-with sources are selected, and the flow submits through `swapAndExecute`.

| Prop | Required | Description |
| --- | --- | --- |
| `config.mode` | Yes | Must be `"deposit"`. |
| `config.deposit` | Yes | Single destination and app-call config. Opens the deposit form directly. |
| `config.prefill.amount` | Optional | Prefills the exact output deposit amount. |
| `config.allowedSourcePairs` | Optional | Restricts the selectable pay-with token/chain pairs. |
| `connectedAddress` | Optional | Wallet address to use. If omitted, the connected wagmi account is used. |
| `embed` | Optional | Defaults to `true`. Set `false` to render as a modal surface. |
| `open`, `onOpenChange`, `defaultOpen` | Optional | Control the modal when `embed={false}`. |
| `onStart`, `onComplete`, `onError`, `onClose`, `onConnectWallet` | Optional | Host app callbacks. Use `onConnectWallet` to route the element CTA to your app-level wallet flow. |

### Deposit config fields (`config.deposit`)

| Field | Type | Required | Description |
| --- | --- | :---: | --- |
| `chainId` | `number` | Yes | Chain ID of the destination network. |
| `tokenAddress` | `` `0x${string}` `` | Yes | Contract address of the destination token on the destination chain. |
| `tokenSymbol` | `string` | Yes | Symbol of the destination token (e.g. `"USDC"`, `"USDT"`). |
| `tokenDecimals` | `number` | Yes | Decimals of the destination token (e.g. `6`, `18`). |
| `executeDeposit` | `function` | Yes | Builder returning the final app call: `(tokenSymbol, tokenAddress, amount, chainId, user) => ExecuteConfig`. |
| `title` | `string` | No | Display title for the deposit (highest priority for headers and UI state). |
| `protocol` | `string` | No | Target protocol/platform name (used in fallback headers and progress labels). |
| `apy` | `string` | No | APY string shown in the card, e.g. `"4.2%"`. |
| `description` | `string` | No | Short description shown in the card. |
| `estimatedTime` | `string` | No | Estimated completion time shown in the UI. |
| `tokenLogo` | `string` | No | Logo URL override for the destination token. |
| `depositTargetLogo` | `string` | No | Protocol/target platform logo (headers and completion screens). |

```ts title="DepositConfig"
interface DepositConfig {

  tokenDecimals: number;

  title?: string;
```

## Modal usage

Render `embed={false}` to invoke Deposit from a button and show the element as a modal.

```tsx showLineNumbers
<NexusOne
  config={depositConfig}
  connectedAddress={address}
  embed={false}
  defaultOpen={false}
/>
```

Control the modal from app state when needed.

```tsx showLineNumbers
const [open, setOpen] = useState(false);

<NexusOne
  config={depositConfig}
  connectedAddress={address}
  embed={false}
  open={open}
  onOpenChange={setOpen}
/>;
```

## Supporting multiple opportunities

`NexusOne` accepts a single `deposit` block so it can run one targeted deposit flow. To offer several opportunities (Aave, Compound, other vaults), build a selector in your parent component and render `NexusOne` with the chosen config:

```tsx showLineNumbers
import { useState } from "react";
import { NexusOne } from "@/components/nexus-one/nexus-one";
import { encodeFunctionData } from "viem";

const OPPORTUNITIES = {
  aave: {
    chainId: 42161,
    tokenSymbol: "USDT",
    tokenDecimals: 6,
    tokenAddress: "0xFd086bC7CD5C481DCC9C85ebE478A1C0b69FCbb9",
    title: "Aave Market",
    protocol: "Aave",
    executeDeposit: (symbol, tokenAddress, amount, chainId, user) => ({
      to: "0x794a61358D6845594F94dc1DB02A252b5b4814aD",
      data: encodeFunctionData({
        /* Aave supply arguments */
      }),
      tokenApproval: {
        toTokenAddress: tokenAddress,
        amount,
        spender: "0x794a61358D6845594F94dc1DB02A252b5b4814aD",
      },
    }),
  },
  compound: {
    chainId: 8453,
    tokenSymbol: "USDC",
    tokenDecimals: 6,
    tokenAddress: "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913",
    title: "Compound Market",
    protocol: "Compound",
    executeDeposit: (symbol, tokenAddress, amount, chainId, user) => ({
      to: "0xc3d688B66703497DAA19211EEdff47f25384cdc3",
      data: encodeFunctionData({
        /* Compound supply arguments */
      }),
      tokenApproval: {
        toTokenAddress: tokenAddress,
        amount,
        spender: "0xc3d688B66703497DAA19211EEdff47f25384cdc3",
      },
    }),
  },
} as const;

export function MultiDepositShowcase({
  address,
}: {
  address?: `0x${string}`;
}) {
  const [selected, setSelected] = useState<keyof typeof OPPORTUNITIES>("aave");

  return (
    <NexusOne
      config={{
        mode: "deposit",
        deposit: OPPORTUNITIES[selected],
      }}
      connectedAddress={address}
    />
  );
}
```
