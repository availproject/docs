In this guide, you will install the required dependencies and add your first Nexus UI Elements component to an existing React project. By the end, your project will be ready for the provider setup in the next step.

## Prerequisites

Before you begin, make sure your project meets the following requirements:

- **React 18+** with a framework like [Next.js](https://nextjs.org), [Vite](https://vitejs.dev), or similar
- **Tailwind CSS v3 or v4** configured in your project
- **A wallet connection library** such as [wagmi](https://wagmi.sh) + [RainbowKit](https://rainbowkit.com) or [ConnectKit](https://docs.family.co/connectkit)
- **Node.js 18+** and a package manager (npm, pnpm, or yarn)

<Callout type="info" title="Why wagmi?">
Nexus UI Elements needs access to the user's connected wallet provider to sign transactions. The library is designed to work with wagmi's hooks, but any EIP-1193 compatible provider will work.
</Callout>

## Step 1 — Install shadcn/ui

Nexus UI Elements is built on [shadcn/ui](https://ui.shadcn.com) primitives. If your project doesn't have shadcn/ui set up yet, follow the official installation guide first.

## Step 2 — Add a Nexus UI Elements component

With shadcn/ui configured, you can now install any Nexus UI Elements component using the CLI. Each component is a self-contained package that brings in its own dependencies and source files.

Run the following command to install a component — here we use `unified-balance` as an example:

```bash
npx shadcn@latest add @nexus-elements/unified-balance
```

This command will:
1. Download the component source files into your project
2. Install any required npm dependencies (like `@avail-project/nexus-core`)
3. Add related UI primitives that the component needs

You can replace `unified-balance` with any available component name. See the full list on the [Components](/docs/nexus/nexus-ui-elements/components/fast-bridge) page.

### Available components

| Component | Install command |
|---|---|
| Unified Balance | `npx shadcn@latest add @nexus-elements/unified-balance` |
| Fast Bridge | `npx shadcn@latest add @nexus-elements/fast-bridge` |
| Deposit | `npx shadcn@latest add @nexus-elements/deposit` |
| Swaps | `npx shadcn@latest add @nexus-elements/swaps` |
| Fast Transfer | `npx shadcn@latest add @nexus-elements/fast-transfer` |
| View History | `npx shadcn@latest add @nexus-elements/view-history` |

## Step 3 — Verify the installation

After running the install command, confirm that the component files were added to your project:

```bash
ls src/components/nexus/
```

You should see the component files along with a `NexusProvider.tsx` file that was automatically scaffolded. This provider is essential — you will configure it in the next step.

<Callout type="info" title="What's the NexusProvider?">
The `NexusProvider` is a React context provider that initializes the Nexus SDK and makes it available to all Nexus UI Elements components in your app. No component will function without it.
</Callout>

## Next step

Your dependencies are installed and the component files are in place. Next, you need to wire up the `NexusProvider` so the SDK can connect to the user's wallet.
