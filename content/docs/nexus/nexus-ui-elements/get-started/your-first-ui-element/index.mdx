You have installed the packages and configured the `NexusProvider`. Now it is time to render a working component. In this guide you will add the **Fast Bridge** component to a page, understand each prop it accepts, and handle its lifecycle callbacks.

## What you will build

By the end of this page you will have a fully functional cross-chain bridge widget embedded in your app. The user will be able to select a token, choose a source chain, and bridge assets — all with built-in progress UI, fee breakdowns, and error handling.

## Step 1 — Install the Fast Bridge component

If you have not already installed the Fast Bridge during the installation step, add it now:

```bash
npx shadcn@latest add @nexus-elements/fast-bridge
```

## Step 2 — Create a page for the bridge

Create a new page (or add to an existing one) that renders the `FastBridge` component. Here is a complete example:

```tsx title="app/bridge/page.tsx" showLineNumbers
"use client";
import { useAccount } from "wagmi";
import { SUPPORTED_CHAINS } from "@avail-project/nexus-core";
import FastBridge from "@/components/nexus/fast-bridge";

export default function BridgePage() {
  const { address } = useAccount();

  if (!address) {
    return <p>Connect your wallet to use the bridge.</p>;
  }

  return (
    <div className="mx-auto max-w-md py-12">
      <FastBridge
        connectedAddress={address}
        onComplete={() => {
          console.log("Bridge complete!");
        }}
        prefill={{
          token: "USDC",
          chainId: SUPPORTED_CHAINS.BASE,
        }}
        onError={(message) => {
          console.error("Bridge error:", message);
        }}
        onStart={() => {
          console.log("Bridge started");
        }}
      />
    </div>
  );
}
```

## Step 3 — Understand the props

Here is a breakdown of every prop passed to `FastBridge` and what it controls:

| Prop | Type | Required | Description |
| --- | --- | --- | --- |
| `connectedAddress` | `Address` | Yes | The wallet address of the connected user. Required for signing transactions. |
| `prefill` | `{ token, chainId, amount?, recipient? }` | No | Pre-populate the bridge form with default values so the user can bridge with fewer clicks. |
| `prefill.token` | `SUPPORTED_TOKENS` | No | The token to bridge (e.g. "USDC", "ETH", "AVAIL"). |
| `prefill.chainId` | `SUPPORTED_CHAINS_IDS` | No | The source chain to bridge from (e.g. SUPPORTED_CHAINS.BASE). |
| `prefill.amount` | `string` | No | A default amount to fill in. |
| `prefill.recipient` | `Address` | No | A recipient address if different from the connected wallet. |
| `onComplete` | `() => void` | No | Called when the bridge transaction completes successfully. Use this to refresh balances or navigate the user. |
| `onStart` | `() => void` | No | Called when the user initiates the bridge. Useful for analytics or disabling other UI. |
| `onError` | `(message: string) => void` | No | Called if the bridge fails. The message string contains a human-readable error description. |

<Callout type="info" title="All props are optional except connectedAddress">
Only `connectedAddress` is required. Everything else has sensible defaults. You can render `<FastBridge connectedAddress={address} />` with zero configuration and it will work.
</Callout>

## Step 4 — Run and test

Start your development server and navigate to the bridge page:

```bash
npm run dev
```

1. Open your app in the browser
2. Connect a wallet using your wallet connection UI
3. The bridge widget should appear with USDC on Base pre-selected
4. Try selecting a different token or chain to verify the form is interactive

<Callout type="warning" title="Testnet first">
When testing bridge transactions, use a testnet configuration to avoid spending real funds. Check the Nexus SDK documentation for supported testnets.
</Callout>

## Swapping components

Every Nexus UI Elements component follows the same pattern: install it, import it, and pass the relevant props. You can replace `FastBridge` with any other component using the same approach.

For example, to show a unified balance view instead:

```tsx
import UnifiedBalance from "@/components/nexus/unified-balance";

// No props required — it reads from the NexusProvider context
<UnifiedBalance />
```

Or to add a swap widget:

```tsx
import Swaps from "@/components/nexus/swaps";

<Swaps
  connectedAddress={address}
  prefill={{ token: "USDC", chainId: SUPPORTED_CHAINS.BASE }}
  onComplete={() => console.log("Swap complete")}
/>
```
